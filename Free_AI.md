## 🎯 Project AI Requirements Analysis

After analyzing the OmniAgentFactory codebase, the project requires:

1. **Tool Calling / Function Calling** — CRITICAL. The agent loop (`backend/agent/loop.py`) uses OpenAI-compatible tool calls to execute swarm agent tasks. The self-evolution system (`backend/core/self_evolution/`) also needs structured JSON output. Tool calling is non-negotiable.

2. **Context Window: 32K+ recommended, 8K minimum** — Agent loops contain full conversation history (ShortTermMemory: 30 entries). Self-evolution prompts can exceed 50K chars (codebase snapshots trimmed to 50K). Gemini 1.5's 1M context is ideal; 128K is comfortable; 32K is workable with truncation.

3. **Medium Throughput (~120 requests/min peak)** — Autonomous evolution cycles every 120s. Swarm multi-agent tasks may fire 3-5 LLM calls concurrently. On average: 30-60 requests/min normal, up to 120/min during peak agent activity.

4. **Streaming Support** — REQUIRED. The agent loop streams tokens via SSE to the frontend. All provider implementations support `stream()` and `stream_async()`.

5. **OpenAI-Compatible API** — PREFERRED. The codebase uses OpenAI-style message format `[{"role": "user", "content": "..."}]`. LiteLLM (used by model_router) translates non-OpenAI formats, but native compatibility reduces friction for direct provider integration.

6. **Latency < 5s per request** — Autonomous loops are time-sensitive. Fast inference (< 3s) is preferred.
