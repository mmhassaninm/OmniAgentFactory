# Watcher Safety & Governance Skill

## Description
Provides the procedural steps for the Watcher Agent to evaluate pending optimization suggestions generated by the main development loop. Ensures deterministic safety without human approval.

## When to Use (Trigger Conditions)
- Triggered whenever a new optimization is posted to the `pending_improvements` collection with status `pending`.

## Procedural Steps
1. **Scope Check**: Inspect the proposed files and text content. Ensure all edits are contained purely within `backend/` directories.
2. **Checkpoint Verification**: Query the `checkpoints` collection to confirm a pre-evolution checkpoint exists for safety rollback.
3. **Test Suite Execution**: Scan the improvement for corresponding test coverage. If none is found, auto-generate a test file using the LLM and run it in an isolated environment.
4. **Regression Run**: Apply the proposed changes to a temporary staging workspace and run the test suite. If any test regressions occur, reject the change.
5. **Novelty Scan**: Calculate similarity metrics against the last 5 approved improvements. Reject duplicate revisions.

## Known Failure Modes & Workarounds
- *Sandbox timeout*: If the regression runner hangs, fallback to a local python syntax validation test before rejecting or approving.
- *LLM test generation fails*: If the test generation produces invalid python code, retry once or fallback to code validation checks.
